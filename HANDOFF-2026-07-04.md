## Comment ça marche (rappel archi)

- **Pipeline** (`src/pipeline/index.ts`) : naia rédige → `extractClaims` →
  Conseil des Mentors (juriste + parlement) vérifie CHAQUE claim contre les
  sources officielles (Légifrance/Assemblée/Sénat via Moulineuse) → arbitre
  **pur code** → statut `answered` (certifié, score≥70) / `insufficient`
  (sources indispo, unknown>50%) / `refused` (contredit ou score<seuil).
- **Gate A = 2 fixes durables** (la clé de la certif) :
  1. *Budget par-phase* : le `deadline` de vérif est calculé APRÈS
     `extractClaims` (avant, il incluait la génération du draft ~44 s et affamait
     la vérif → tout partait en `unknown`).
  2. *Calibration tool-calls* : interdiction d'inventer `query_by:'title'` sur
     `search_legal_texts` (la collection `textes_juridiques` n'a pas de champ
     `title` → 404) ; repli sur `search_legal_texts`(query seul)+
     `get_pastilled_article` quand `search_recipes` timeout ; pas d'alias SQL non
     défini. Instructions dans `src/mastra/agents/{naia,mentor-juriste}/instructions.md`.
- **MOCK = 2 niveaux** :
  - *tool* (`MOCK_MOULINEUSE=1`, `src/mastra/mock.ts` + `mcp.ts`) : sert
    `fixtures/moulineuse/<tool>.json` au lieu d'appeler la Moulineuse.
  - *réponse* (`DEMO_REPLAY=1`, `src/pipeline/index.ts`) : rejoue un
    `PipelineResult` certifié capturé (`fixtures/demo/<slug-question>.json`),
    déterministe. Capture via `CAPTURE_DEMO=1` sur un run live `answered`.
- **canvakit** : `@canvakit/core`+`@canvakit/export` en `file:../../canvakit/packages/*`
  (override `workspace:*` + `allowBuilds.canvas:false` dans `pnpm-workspace.yaml`
  local), thème **DSFR inline** (Bleu France `#000091` / Rouge Marianne `#E1000F`,
  SANS designkit), fallback local (`engine: canvakit|local`). Tool

  les agents spawnés ne font PAS de git. Arbre partagé → committer par feature,
  vérifier le scope avant `git add`.

  gitignoré ; `.env.example` = référence publique (valeurs vides).

  Relancer le serveur naia à la main (cf TL;DR).
